# 智能划重点方法论

## 一、核心原则

### 1. 尊重中文语言习惯

**固定搭配不可拆分**

| 错误 ❌ | 正确 ✅ |
|---------|---------|
| 高亮"大"（来自"大家"） | 高亮"大家" |
| 高亮"很"（来自"很多"） | 高亮"很多" |
| 高亮"会"（来自"不会"） | 高亮"不会" |
| 高亮"能"（来自"能建起来"） | 高亮"能建起来" |

### 2. 每行最多2个高亮

**避免过度标记**

```
❌ 错误：然后很多细心的朋友可以注意到
        ↑高亮4个词：很、细心、朋友、可以

✅ 正确：然后很多细心的朋友可以注意到
        ↑高亮2个词：很多、细心
```

### 3. 考虑上下文

**不孤立判断单行**

同一个词在不同上下文中的重要性不同：
- "小朋友"在谈论儿童设施时是核心词 → 高亮
- "小朋友"在泛指人群时不重要 → 不高亮

## 二、技术实现

### 2.1 中文分词

```python
import jieba
import jieba.posseg as pseg

# 基础分词
words = jieba.cut("然后很多细心的朋友可以注意到")
# 结果：['然后', '很多', '细心', '的', '朋友', '可以', '注意', '到']

# 带词性标注
words_pos = pseg.cut("这个座位非常宽敞")
# 结果：[('这个', 'r'), ('座位', 'n'), ('非常', 'd'), ('宽敞', 'a')]
```

### 2.2 固定搭配合并

**问题**：jieba 可能把固定搭配拆开

```python
# jieba默认分词
jieba.cut("能建起来")
# → ['能', '建', '起来'] ❌

# 解决方案：固定搭配后处理
FIXED_COLLOCATIONS = {'能建起来', '很多', '不会', ...}

def segment_with_collocations(text):
    words = list(jieba.cut(text))
    result = []
    i = 0
    while i < len(words):
        # 尝试合并后续N个词，检查是否匹配固定搭配
        found = False
        for length in range(4, 0, -1):
            phrase = ''.join(words[i:i+length])
            if phrase in FIXED_COLLOCATIONS:
                result.append(phrase)
                i += length
                found = True
                break
        if not found:
            result.append(words[i])
            i += 1
    return result
```

### 2.3 关键词评分

**多维度评分系统**

```python
def calculate_keyword_score(word, context):
    score = 0.0

    # 1. 禁用词直接返回0
    if word in STOP_WORDS:
        return 0.0

    # 2. 单字词不高亮（除非是固定搭配的一部分）
    if len(word) == 1 and word not in FIXED_COLLOCATIONS:
        return 0.0

    # 3. 类别优先级
    for category, info in KEYWORD_CATEGORIES.items():
        if word in info['keywords']:
            score += info['priority'] * 10

    # 4. 固定搭配加分
    if word in FIXED_COLLOCATIONS:
        score += 15

    # 5. 长度加分（更长的词可能更重要）
    if len(word) >= 2:
        score += len(word) * 2

    # 6. 词性加分（名词和形容词）
    if context.get('pos') in ['n', 'a', 'ad', 'an']:
        score += 5

    return score
```

**评分示例**

| 词 | 类别得分 | 固定搭配 | 长度 | 词性 | 总分 |
|----|----------|----------|------|------|------|
| 非常 | 40 (强调词×4) | - | 4 | d | **44** |
| 高铁 | 30 (核心名词×3) | - | 4 | n | **39** |
| 能建起来 | 20 (固定搭配×2) | +15 | 8 | v | **43** |
| 可以 | 10 (能愿动词×1) | - | 4 | v | **14** |
| 然后 | - | - | - | - | **0** (禁用词) |

## 三、关键词分类体系

### 3.1 强调词（优先级4）

用于表达程度、强调语气

```python
'非常', '十分', '特别', '最', '超级', '极其'
```

**特点**：高频出现，立即引起注意

### 3.2 核心名词（优先级3）

视频主题相关的实体

```python
'高铁', '座位', '空间', '杂志', '质量', '工程', '小朋友', '窗边'
```

**特点**：视频内容的主角，需根据视频主题调整

### 3.3 正面形容词（优先级3）

描述品质、特征的积极词汇

```python
'宽敞', '有利', '伟大', '细心'
```

**特点**：烘托氛围，传递正面情绪

### 3.4 固定搭配（优先级2）

中文习惯用语、成语、动词短语

```python
'能建起来', '群山环绕', '有一种', '无论是', '不会'
```

**特点**：语义完整，不可拆分

### 3.5 能愿动词（优先级1）

表示可能、许可、意愿

```python
'可以', '能够', '应该'
```

**特点**：较弱，通常只在没有更高优先级词时才选

## 五、上下文感知

### 5.1 前后行关联

```python
def select_highlights_for_line(text, prev_line='', next_line=''):
    # 分析当前行
    words = segment_with_collocations(text)

    # 考虑前后行的关键词，避免重复高亮
    # 例如：
    # 前行：这个座位非常宽敞
    # 当前：脚呢是非常有这种空间性的
    # → 如果"非常"已在前行高亮，当前行可降低"非常"权重

    candidates = []
    for word in words:
        context = {
            'prev_line': prev_line,
            'next_line': next_line,
            'text': text,
        }
        score = calculate_keyword_score(word, context)
        if score > 0:
            candidates.append((word, score))

    # 按得分排序，选择前2个
    candidates.sort(key=lambda x: x[1], reverse=True)
    return [word for word, score in candidates[:2]]
```

### 5.2 主题一致性

**根据视频主题调整关键词列表**

## 六、ASS字幕生成

### 6.1 文件结构

```ass
[Script Info]
Title: 智能划重点字幕
ScriptType: v4.00+
WrapStyle: 0
ScaledBorderAndShadow: yes
# ⚠️ 不要设置 PlayResX/PlayResY，让字幕自适应视频分辨率

[V4+ Styles]
Format: Name, Fontname, Fontsize, PrimaryColour, ...
Style: Default,PingFang SC,24,&H00FFFFFF,...

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,00:00:00.00,00:00:03.60,Default,,0,0,0,,普通文字{\fs28\c&H00FFFF&\b1}高亮词{\r}继续
```

### 6.2 内联样式标记

```python
def highlight_keywords(text, keywords):
    """为关键词添加ASS样式标记"""
    result = text
    # 按长度倒序，避免短词替换影响长词
    for kw in sorted(keywords, key=len, reverse=True):
        # 黄色、28pt、加粗
        highlight_tag = r'{\fs28\c&H00FFFF&\b1}' + kw + r'{\r}'
        result = result.replace(kw, highlight_tag)
    return result
```

**样式代码含义**

| 代码 | 含义 |
|------|------|
| `\fs28` | 字号28pt（默认24pt） |
| `\c&H00FFFF&` | 黄色（BGR格式：蓝0 绿FF 红FF） |
| `\b1` | 加粗（0=正常，1=粗体） |
| `{\r}` | 重置为默认样式 |

### 6.3 时间戳转换

SRT → ASS 时间格式转换

```python
# SRT: 00:00:12,340 --> 00:00:15,670
# ASS: 0:00:12.34,0:00:15.67

def srt_to_ass_time(srt_time):
    """
    SRT: 00:00:12,340
    ASS: 0:00:12.34
    """
    # 去掉前导0，逗号改为点，去掉最后一位毫秒
    return srt_time.lstrip('0').replace(',', '.')[:-1]
```

**场景**：用户反馈"大家"被拆分为"大"

```python
keywords = ['大', '可以']
```

```python
# 1. 添加固定搭配
FIXED_COLLOCATIONS.add('大家')

keywords = ['大家', '可以']
```

## 八、常见问题

### Q1: 为什么"然后"、"或者"不高亮？

这些是连接词，属于功能词，不承载实际语义。

**反例**：如果高亮这些词
```
然后很多细心的朋友可以注意到
↑↑↑  全是黄色，核心词"细心"反而不突出
```

### Q2: 如何处理专有名词？

**方法1**：添加到核心名词列表
```python
KEYWORD_CATEGORIES['核心名词']['keywords'].extend([
    '钉钉AI', 'GPT-4', '剪映'
])
```

**方法2**：使用 jieba 用户词典
```python
import jieba
jieba.add_word('钉钉AI')  # 确保不被拆分
```

### Q3: 能否根据语气自动调整？

可以！例如：
- 正面情绪 → 多高亮形容词（"宽敞"、"伟大"）
- 中性叙述 → 多高亮名词（"座位"、"空间"）

```python
def detect_sentiment(text):
    positive_words = ['好', '棒', '优秀', '伟大']
    return any(w in text for w in positive_words)

if detect_sentiment(text):
    # 提高形容词权重
    KEYWORD_CATEGORIES['正面形容词']['priority'] += 1
```

## 九、性能优化

### 9.1 jieba 模型缓存

```python
import jieba

# 首次加载较慢（~0.2秒）
jieba.cut("测试文本")

# 后续调用直接使用缓存
# Loading model from cache /tmp/jieba.cache
```

### 9.2 批量处理

```python
# ❌ 低效：逐行处理
for line in subtitles:
    words = jieba.cut(line)
    # ... 处理

# ✅ 高效：一次性加载全部文本
all_text = '\n'.join([sub['text'] for sub in subtitles])
jieba.cut(all_text)  # 预热缓存
for line in subtitles:
    words = jieba.cut(line)  # 快速
```

## 十、验证清单

在生成最终视频前，检查：

- [ ] 固定搭配是否完整（"大家"、"很多"、"不会"）
- [ ] 每行高亮数量 ≤ 2
- [ ] 禁用词是否正确过滤（"然后"、"或者"）

- [ ] 字体大小是否正确（ASS无PlayResX/Y）
- [ ] 时间戳格式是否正确（ASS格式）
