# Changelog

## [3.0.0] - 2026-03-25

### 🆕 Added - 字幕可选字体

烧录前新增字体三选一流程，生成预览截图供用户可视化选择。

**核心特性**：
- ✅ 三种字体样式 — 苹方 PingFang / 中华薪火体（带底条）/ 思源宋体 Bold（衬线描边）
- ✅ 预览截图 — 提取第一帧，分别用3种字体烧录预览图，用户对比选择
- ✅ 分辨率自适应 — 1080p FontSize=50，720p FontSize=24，自动检测
- ✅ 烧录后自动清理 — 删除 json/srt/ass 等中间文件，只保留原视频和成品

**字体样式**：

| 字体 | 特点 | ASS 关键参数 |
|------|------|-------------|
| 苹方 PingFang | 圆润现代，黑色描边 | BorderStyle=1, Outline=3 |
| 中华薪火体 | 手写风格，50%透明黑底条 | BorderStyle=3, Outline=8 |
| 思源宋体 Bold | 衬线加粗，黑色描边 | Bold=-1, Outline=3 |

### 🔧 Fixed

- 字体2底条对齐：用 `Outline` 实现四周均匀 padding（不用 `Shadow`，Shadow 只往右下偏移）
- ASS 字号：CSS 24px 在 1080p PlayRes 下太小，调整为 50

### 📝 Updated

- `字幕/SKILL.md` — 完整字体三选一流程、ASS 样式定义、分辨率自适应表、清理规则
- `剪辑/SKILL.md` — 新增剪辑相关更新
- `自更新/SKILL.md` — 新增反馈记录（底条对齐、字号、用字纠错）

## [2.0.0] - 2026-01-26

### 🆕 Added - 智能划重点功能

基于真实用户（@Kora）两天密集使用反馈，新增智能字幕高亮系统。

**核心特性**：
- ✅ 中文语境识别 - 固定搭配不拆分（"大家"、"很多"、"不会"、"能建起来"）
- ✅ 精准控制 - 每行最多2个高亮，避免过度标记
- ✅ 上下文感知 - 结合前后行选择最重要的词
- ✅ 多维度评分 - 类别、词性、长度、固定搭配综合评分
- ✅ 智能过滤 - 自动过滤90+功能词（"然后"、"或者"、"的"、"了"）

**新增文件**：
```
智能划重点/
├── SKILL.md                    # Skill 配置
├── README.md                   # 使用文档
├── smart_highlight.py          # 核心算法实现
└── tips/
    └── 智能划重点方法论.md     # 详细技术文档
```

**技术栈**：
- jieba - 中文分词引擎
- jieba.posseg - 词性标注
- FFmpeg (libass) - ASS字幕烧录

### 🐛 Fixed

**字体大小兼容性问题**：
- **问题**：v1版本在不同分辨率视频上字幕大小不一致
- **原因**：ASS文件中设置了固定的 `PlayResX: 1920` 和 `PlayResY: 1080`
- **修复**：移除固定分辨率设置，让字幕自适应视频分辨率

**词语边界识别错误**：
- **问题**：简单字符串匹配导致"大家"被拆分为"大"高亮
- **修复**：使用 jieba 分词 + 固定搭配后处理，完整识别词语边界

**过度高亮问题**：
- **问题**：一行出现4-5个高亮词，影响视觉体验
- **修复**：严格限制每行最多2个高亮，按优先级排序选择

**转录错误**：
- **问题**："如果有一种的小朋友" → 多余"的"字
- **修复**：添加自动纠错规则 `fix_transcription_errors()`

### 📝 Changed

**主 README 更新**：
- 新增"智能划重点"功能介绍
- 更新工作流程图，加入智能划重点步骤

## 用户反馈驱动的开发

### 真实使用案例

**用户**: @Kora
**使用场景**: 高铁 vlog 视频剪辑
**反馈周期**: 2天
**迭代次数**: 3次重大改进

### 关键反馈时刻

#### 反馈 #1 - 词语边界问题
> "大：这个词的左右两边的这一行文字是有上下文的，'大家'是一个词，只化重点在大，中文语义里不合适"

**影响**: 促使我们引入 jieba 分词引擎，从简单字符串匹配升级到语义级识别

#### 反馈 #2 - 过度高亮
> "很、细心、朋友、可以 - 太多高亮了，一行标记 ≤ 两处"

**影响**: 添加严格限制机制，按优先级排序选择 Top 2

#### 反馈 #3 - 固定搭配
> "'不会'是固定搭配，'能建起来'是常见动词短语，应该一起高亮"

**影响**: 建立固定搭配词典，识别90+常见中文搭配

#### 反馈 #4 - 字体大小
> "V2版本的字幕整体都比之前小特别多！"

**影响**: 发现并修复 ASS 分辨率设置bug

### 设计哲学

**"用户反馈 > 技术炫技"**

我们不追求：
- ❌ 最先进的 NLP 模型
- ❌ 最复杂的算法架构
- ❌ 最多的功能选项

我们追求：
- ✅ **真实场景下好用** - 基于实际使用反馈迭代
- ✅ **尊重用户习惯** - 理解中文语境，而非机械匹配
- ✅ **保持简单** - 90%场景开箱即用，10%场景可配置

## 贡献指南

### 如何提供反馈

**方式1: 在 Claude Code 中**
```
/videocut:自更新

告诉 AI: "我觉得'XXX'不应该高亮，因为..."
```

**方式2: GitHub Issues**
```markdown
## 问题描述
[具体描述问题]

## 复现步骤
1. 准备字幕文件 xxx.srt
2. 运行 /videocut:智能划重点
3. 发现问题：...

## 期望行为
我希望...

## 截图
[可选]
```

### 反馈的黄金标准

参考 @Kora 的反馈模式：

✅ **好的反馈**：
- 具体：指出哪一行、哪个词有问题
- 有上下文：提供前后行内容
- 有建议：说明期望的行为
- 有截图：视觉问题配截图

❌ **不好的反馈**：
- "不好用" - 太模糊
- "有bug" - 没有具体信息
- "不如剪映" - 缺少对比细节
